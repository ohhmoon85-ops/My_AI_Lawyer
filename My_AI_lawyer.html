프로젝트 개요

본 프로젝트는 법제처 국가법령정보 오픈 API와 Claude AI를 결합하여, 사용자의 질문에 대해 실제 대한민국 법령과 판례를 근거로 답변하는 지능형 법률 상담 서비스를 구축합니다. AI의 단순 추론이 아닌 실시간 법령 데이터를 조회하여 신뢰도 높은 답변을 생성하는 RAG 아키텍처를 채택합니다.

📜

참조한 법률명과 조항 번호를 명확히 표시하여 신뢰성 확보.
💬
채팅 인터페이스
일상 언어로 질문 가능한 웹 기반 실시간 채팅 UI.

모바일 접근성 확보를 위한 Telegram API 연동 (선택).

05

기술 검토 사항

개발 착수 전 반드시 숙지해야 할 3가지 기술적 주의사항입니다.

CRITICAL

Vercel 서버리스 타임아웃 → Streaming 필수

Vercel의 서버리스 함수는 기본 10초, Hobby 플랜 최대 60초의 타임아웃이 있습니다. 법령 API 호출 + Claude 추론 시간이 이를 초과할 경우 504 Gateway Timeout이 발생합니다.

// app/api/chat/route.ts — Streaming 방식
return new Response(stream, {
  headers: { 'Content-Type': 'text/event-stream' }
});

→ Claude Code에게 ReadableStream + TransformStream 방식으로 API Route를 작성해달라고 명시적으로 요청하세요.

OPTIONAL

텔레그램 Webhook — /api/telegram 엔드포인트

Vercel 배포 환경에서 텔레그램 봇은 Polling 방식이 아닌 Webhook 방식으로만 안정적으로 동작합니다. Polling은 상시 실행 서버가 필요하기 때문입니다.

// Webhook 등록 (배포 후 1회 실행)
POST https://api.telegram.org/bot{TOKEN}/setWebhook
  url: "https://your-domain.vercel.app/api/telegram"

→ /api/telegram Route는 텔레그램 메시지를 수신한 뒤 동일한 AI 파이프라인(/api/chat)을 재활용하도록 구현하면 코드 중복을 최소화할 수 있습니다.

IMPORTANT

법령 API 응답 XML/JSON 파싱 & 정규화

국가법령정보 API의 응답은 중첩 XML 또는 복잡한 JSON 구조입니다. 이를 그대로 Claude 프롬프트에 삽입하면 토큰이 낭비되고 답변 품질이 저하됩니다. 정규화 함수가 핵심 품질 결정 요소입니다.

// 정규화 목표 형태 (Claude에게 전달)
법률명: 근로기준법 제56조 (연장·야간·휴일근로)
내용: 사용자는 연장근로에 대해 통상임금의
       100분의 50 이상을 가산하여 지급...

→ xml2js 또는 내장 DOMParser를 활용하여 법률명·조문번호·내용만 추출하는 parseStatute() 함수를 Phase 02에서 우선 완성하세요.

보안 및 주의사항

🔐 API 키 노출 방지

.env.local 파일은 반드시 .gitignore에 포함하여 GitHub에 업로드되지 않도록 주의하십시오. Vercel 배포 시 환경 변수는 대시보드에서 별도 설정해야 합니다.

⚖️ 법적 면책 조항 필수 표시

UI 하단에 "본 서비스의 답변은 참고용이며, 실제 법적 효력을 갖지 않습니다. 중요한 법률 문제는 전문 변호사와 상담하시기 바랍니다." 문구를 반드시 명시해야 합니다.

⚡ API 사용량 관리

Claude API 및 국가법령정보 API 모두 사용량 제한이 있습니다. 프로덕션 환경에서는 Rate Limiting 미들웨어와 캐싱 전략(Redis 등)을 도입하여 비용을 관리하세요.

📋 가드레일 설정

시스템 프롬프트에 "검색된 법령 데이터 내에서만 답변하고, 관련 법령이 없거나 불확실한 경우 모른다고 답변할 것"이라는 지침을 반드시 포함하여 AI 환각을 방지하세요.
